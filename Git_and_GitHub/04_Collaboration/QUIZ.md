## Section 4: Forking

### Question 6 (‚≠ê‚≠ê Intermediate)
**What is the difference between forking and cloning?**

A) They're the same  
B) Fork creates a copy on GitHub; clone downloads to your computer  
C) Fork is for private repos only  
D) Clone creates a copy on GitHub

<details>
<summary>Click to reveal answer</summary>

**Answer: B - Fork creates a copy on GitHub; clone downloads to your computer**

**Explanation**: Forking creates your own copy of a repository on GitHub (in your account), while cloning downloads any repository to your local machine.

</details>

## Section 6: Best Practices

### Question 9 (‚≠ê‚≠ê Intermediate)
**How do you link a PR to an issue?**

A) Use #issue-number in the PR description  
B) Use keywords like "Fixes #123" or "Closes #456"  
C) Manually close the issue after merging  
D) It's not possible

<details>
<summary>Click to reveal answer</summary>

**Answer: B - Use keywords like "Fixes #123" or "Closes #456"**

**Explanation**: Keywords like "Fixes", "Closes", or "Resolves" followed by #issue-number will automatically close the issue when the PR is merged.

</details>
